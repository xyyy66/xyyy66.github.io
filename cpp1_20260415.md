## `open, write, read`

`open`函数的原型为

```c
int open(char* path, int mode)
```

假设我们传进去的`path`是`/home/user1/a.txt`，操作系统会从根目录开始，逐级查找目录项，直到最终找到 `a.txt`的inode number.

具体的细节：假设根目录的 inode number 是2，操作系统会访问 inode table，找到编号为2的条目。这个条目中有指向对应 inode 的指针，告诉我们根目录的内容存储在磁盘的哪些物理块上，随后操作系统会把这个物理块读入内存，这里面存储的信息有根目录下文件的文件名，inode 编号，文件类型。

目录也是文件，其在磁盘上存储的称为目录项。

之后内核检查权限，检查通过后为该进程分配一个文件描述符，并在进程中的文件描述符表中建立索引，指向全局的打开文件表。打开文件表是全局唯一的，记录了全系统所有被打开的文件状态，里面的记录了文件的读写指针偏移量，引用计数（有多少个 fd 指向这个条目），打开模式，指向 inode 的指针。


`read`函数的原型为

```c
int read(int fd, char* buf, size_t count)
```
操作系统不会直接去读磁盘，首先会查看页缓存是否有了对应的文件数据（如果为被标记为脏页），如果缓存命中，直接把数据从页缓存拷贝到内存中，未命中就会进行磁盘 IO ， 首先是把数据从磁盘拷贝到页缓存，然后再把页缓存中的数据拷贝的内存中。这里其实是 CPU 发出 DMA 请求，让磁盘控制器把数据从磁盘搬运到页缓存。后面的拷贝是有 CPU 完成的。

`write`函数原型为：
```c
int wtite(inf fd, char* buf, size_t count)
```

将数据从`buf`拷贝到内核页缓存中，`write` 就返回了。内核会把此数据标记为脏页，内核会等待一段时间或者等脏页到达一定比例，会又内核线程统一持久化这些数据。

## 数据如何从物理介质被读出来？

在调用了`read(fd, buffer, size)`之后，文件系统首先会帮我们通过查找文件描述符表和打开文件表找到这个文件对应的inode, inode 记录了该文件占用了那些逻辑块（LBA）。内核会首先检查页缓存中是否有这些数据的缓存。如果没有的话，内核会通知磁盘控制器。

逻辑块是操作系统层面的说法，存储内部并不使用逻辑块来划分数据。所以对于不同的存储，其内部会有不同的处理。

+ *磁盘*：计算LBA对应的柱面，磁头，扇区
+ *SSD*： 控制器会通过FTL将LBA映射到具体的物理页

下面是两种硬件不同的搬运方式：

+ *HDD*：
    - 寻道：磁臂移动，让磁头对准数据所在的磁道
    - 旋转：盘片告诉旋转，直到目标扇区转到磁头下方
    - 电磁感应：磁头实际上是一个线圈，通过敢生电流的方式代表0和1
    - 信号转换：磁盘控制器将这些电流波动放大并数字化为二进制序列


+ *SSD*：
    有点复杂...

在获得了位流之后：
+ 纠错：控制器会检查校验码
+ DMA：为了不占用CPU资源，磁盘控制器会通过 DMA 引擎直接把数据写进内存
+ 中断信号：完成搬运之后，磁盘会触发一个硬件中断，CPU会去处理


需要注意的是，无论是HDD还是SSD，无论是读还是写，顺序读取的效率都要高于随机读取。
